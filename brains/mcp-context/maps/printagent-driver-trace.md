# PrintAgent Driver Trace

## Purpose
`PrintAgent` is a local client-side printing driver used by MCP to print documents generated by the web backend, mainly:
- invoices
- tickets
- printable documents with QR

The current implementation is packaged as a PyInstaller executable and runs on the user's machine, not on the MCP server.

## Observed Runtime Contract
- HTTP listener: `http://127.0.0.1:9100`
- Primary endpoint observed: `POST /`
- Log evidence shows the driver receiving a JSON payload with:
  - `texto`
  - `lineas`
  - `logo`
  - `qr`
  - `cortar`

## Render Pipeline
1. MCP or another client submits printable content to the local HTTP driver.
2. The driver parses the payload and prepares the document body.
3. QR markers are replaced with generated QR images.
4. The document is rendered locally into a tall bitmap.
5. The bitmap is split into segments.
6. Each segment is rasterized to ESC/POS bands.
7. The driver sends the bands to the configured printer.

## Printer Behavior
- Observed printer: `POS-80C`
- Print mode: ESC/POS raster
- Segmentation behavior:
  - ticket is split into multiple images
  - each image is sent as multiple raster bands
  - logs show successful completion with HTTP 200

## What Is Confirmed
- The driver already works for receipts/tickets/invoices.
- The driver is local and client-side.
- It prints by rasterizing the rendered content.
- QR replacement is part of the current contract.
- Cutting is supported through the `cortar` flag.

## What Is Not Yet Confirmed
- A dedicated label-print mode for barcode labels.
- A stable, documented ZPL contract.
- Any server-side spool or queue beyond the local HTTP bridge.

## Extension Strategy For Labels
For future barcode labels, the cleanest approach is to add a second print mode instead of mixing it with ticket rendering:
- `mode: ticket`
- `mode: label`

That would allow:
- ticket/invoice logic to remain stable
- label generation to use its own template and sizing rules
- barcode printers or alternate layouts to be isolated

## Risks / Constraints
- Because the current artifact is packaged, source-level refactors require the original Python project.
- The local HTTP bridge is a dependency for MCP printing flows, so any label extension should preserve backward compatibility.
- Printer names and raster behavior appear environment-specific.
